## 数据文件说明

本 Skill 的参考数据分五个文件，所有价格和信息均基于网络实时采集（30+数据源）：

| 文件 | 内容 | 数据来源 | 何时读取 |
|------|------|---------|---------|
| `references/cities_data.md` | 七城景点(15+含实时票价)、美食(含实时价格)、推荐路线、消费档次 | 官方景区网站、携程、本地宝、知乎攻略、搜狐旅游 | 用户问景点/美食/路线/预算时 |
| `references/pitfalls_and_tips.md` | 深度避坑手册、城市专属坑点表格、通用实用 Tips | 花路网、大众点评、携程游记、真实游客反馈 | 用户问避坑/注意事项时 |
| `references/recommendations.md` | 各城最值得去的3处、隐藏宝藏、精选餐厅推荐 | 综合多平台高分评价 | 用户问推荐/隐藏景点时 |
| `references/recommended_shops.md` | **七城推荐店铺手册**（含具体店名/地址/人均/必点菜） | **大众点评2025必吃榜**（长沙45家/深圳84家）、搜狐美食年终榜、南方+韶关旅游、清远旅游官方47家、携程美食林 | 用户问"去哪家店""具体推荐餐厅""本地人推荐哪里吃"时 |
| `references/hidden_attractions.md` | **七城隐藏/小众景点全收录**（含具体地址/门票/出片指数） | **小红书爆款攻略**（2025-2026年）+搜狐旅游+车主指南+网易号+清远本地宝，25+平台实采 | 用户问"人少景美""小众秘境""隐藏宝藏""当地人才知道的地方"时 |

> **数据采集时间**：2026年4月，价格为当前市场参考价，实际以景区/餐厅为准
> **景点数据说明**：隐藏景点来自小红书爆款攻略、搜狐旅游车主指南（2025-2026年），每城收录8-10个小众秘境，含具体地址和门票；长沙深圳额外收录文创/工业/欧式等特色景点；全部景点均标注适合季节和推荐理由

## 执行逻辑

### Step 1 — 识别意图与城市

分析用户问题，提取：
- **目标城市**（支持单城 / 多城 / 全部七城）
- **查询类型**：景点 / 美食 / 避坑 / 推荐 / 消费档次 / 综合攻略

### Step 2 — 选择数据源

根据查询类型读取对应 references 文件（见上表）。

- 用户问"景点/景区"→ 读 `cities_data.md`
- 用户问"美食/吃什么"→ 读 `cities_data.md` + `recommended_shops.md`
- 用户问"哪家店/推荐餐厅/具体店铺"→ 优先读 `recommended_shops.md`
- 用户问"避坑/注意事项"→ 读 `pitfalls_and_tips.md`
- 用户问"推荐/宝藏/隐藏"→ 读 `recommendations.md`
- 用户问"人少景美/小众/秘境/隐藏/当地人才知道"→ **优先读 `hidden_attractions.md`**，备选 `recommendations.md`
- 用户要求"综合攻略"→ 五个文件全读取

### Step 3 — 构建回答结构

按以下结构输出（按用户需求选择全部或部分模块）：

```
## [城市名] 旅游攻略

### 🗺️ 必去景点（≥15个）
（列出景点，注明门票/最佳时间/适合人群）

### 🍜 特色美食
（分类：街头小吃 / 正餐推荐 / 网红打卡 / 伴手礼）

### ⚠️ 避坑提示
（表格形式：坑点 | 详细说明 | 破解方法）

### 💡 推荐精选
（最值得去的3处 + 隐藏宝藏 + 精选餐厅）

### 🌿 隐藏秘境（小众人少景美）
（当用户问"小众/秘境/人少景美/当地人才知道"时，优先输出此模块：
列出8-10个小众景点，注明具体地址、门票、出片指数/适合季节、推荐理由）

### 💰 消费档次分级
（表格：档次 | 日均花费 | 住宿推荐 | 餐饮建议）

### 📅 推荐行程
（2日/3日/主题路线）
```

### Step 4 — 数据完整性验证

输出景点时，每座城市必须包含 **不少于15个景点**。
若检索结果不足，通过运行以下命令补充验证：

```bash
python scripts/city_travel_search.py --count
```

### Step 5 — 消费档次输出规范

消费档次必须包含三个层级：

| 档次 | 日均花费区间 | 必须包含的字段 |
|------|------------|-------------|
| 平价（穷游） | 当地标准最低消费 | 住宿/餐饮/景点 |
| 中档 | 正常游客平均消费 | 同上 |
| 高档（精品游） | 高品质体验消费 | 同上 |

---

## 输出质量要求

1. **景点数量**：每城不少于 15 个，注明门票价格及最佳时间
2. **美食描述**：包含特色描述 + 推荐店铺/区域 + 价格参考
3. **避坑提示**：使用表格形式，覆盖该城市最典型的 4-6 个坑
4. **消费档次**：三档必须给出具体金额区间，不可含糊
5. **实用性**：每条信息需要对读者有实际帮助，避免空泛描述

## 快速检索命令参考

```bash
# 查询某城市的美食
python scripts/city_travel_search.py --city 郴州 --type food

# 查询某城市的避坑信息
python scripts/city_travel_search.py --city 长沙 --type pitfall

# 查询所有城市的推荐景点
python scripts/city_travel_search.py --type recommend

# 验证景点数量是否达标（每城≥15个）
python scripts/city_travel_search.py --count

# 全部七城综合信息
python scripts/city_travel_search.py --type all

# 关键词搜索（如搜索温泉相关内容）
python scripts/city_travel_search.py --keyword 温泉
```
